Deploy Applications from Local Git Repo with ArgoCD#

This article walks through the entire process of using ArgoCD to deploy an application from a GitLab repo in my home lab (a restricted-access environment).

Home Lab Env#

The following services are running in the home lab:

GitLab server:#

http://192.168.1.4:8080/

K8s Cluster#

I have a 3-node k8s cluster: 1 master, 2 workers

❯ kubectl cluster-info

CoreDNS is running at https://192.168.1.101:6443/api/v1/namespaces/kube-system/services/kube-dns:dns/proxy

To further debug and diagnose cluster problems, use 'kubectl cluster-info dump'.

Prepare the Repo#

Create GitLab repo#

Git repo name: argocd-demo-gl

http://192.168.1.4:8080/yahoon/argocd-demo-gl // public

Clone to local dir: yahoon/github

❯ git clone ssh://git@192.168.1.4:2222/yahoon/argocd-demo-gl.git
Cloning into 'argocd-demo-gl'...

Populate the Git Content#

Directory structure:

❯ tree .
.
├── container-image
│   └── argocd-demo
│       ├── Containerfile
│       ├── index.html
│       └── README.md
├── k8s-apps
│   └── argocd-demo
│       ├── deployment.yaml
│       └── service.yaml
└── README.md

Content overview

  • container-image: holds the image, named argocd-demo-gl
    • Serves an http/80 page displaying Hello, This is Yahoon's ArgoCD Demo from Local GitLab!
  • k8s-apps: k8s resources to deploy, mainly the argocd-demo-go service

Do git commit & push

All the code is accessible on https://github.com/yahoon/argocd-demo-gl for your reference.

Prepare Image Repo#

Create a public repo on Quay: https://quay.io/repository/yahoon/argocd-demo-gl

Prepare the Image#

Name it argocd-demo-gl

❯ cd container-image/argocd-demo/
❯ podman build -t quay.io/yahoon/argocd-demo-gl:v1 .

Test image locally

❯ podman run --rm -d  -P quay.io/yahoon/argocd-demo-gl:v1

❯ podman ps
CONTAINER ID  IMAGE                             COMMAND               CREATED        STATUS        PORTS                  NAMES
226a6d5f6589  quay.io/yahoon/argocd-demo-gl:v1  nginx -g daemon o...  2 seconds ago  Up 2 seconds  0.0.0.0:44137->80/tcp  intelligent_banzai

❯ curl localhost:44137
Hello, This is Yahoon's ArgoCD Demof rom Local GitLab!

Push the image

❯ podman login -u='yahoon' -p='yahoon-quay-password' quay.io
❯ podman push quay.io/yahoon/argocd-demo-gl:v1

Deploy the App via ArgoCD UI#

Create app#

Login to Argo UI: https://192.168.1.101:31157/

Click + NEW APP, fill in:

General Section

  • Application Name: argocd-demo-gl
  • Project: default
  • Sync Policy: Manual Sync Options (expand the section)
    • Check Auto-Create Namespace

Source Section

  • Repository URL: http://192.168.1.4:8080/yahoon/argocd-demo-gl.git
  • Revision: HEAD
  • Path: k8s-apps/argocd-demo

Destination Section

  • Cluster URL: select in-cluster (https://kubernetes.default.svc)
  • Namespace: argocd-ademo-gl

Click CREATE in the top right corner

Sync app#

Open the app page, click sync, wait a moment — it succeeds.

Check from the CLI:

Test the app#

❯ ❯ kubectl run curl-test -n argocd-demo-gl --rm -it --image=quay.io/curl/curl --restart=Never -- \                                                           
  curl http://argocd-demo-gl.argocd-demo-gl.svc.cluster.local 
// Should return "Hello, This is Yahoon's ArgoCD Demo".

Port forwarding

❯ kubectl port-forward -n argocd-demo-gl svc/argocd-demo-gl 8081:80

Visit http://localhost:8081/ in your browser

Summary#

In short, ArgoCD directly deploys applications to k8s from a git repo — all changes to the application must go through git. This is the essence of GitOps!
