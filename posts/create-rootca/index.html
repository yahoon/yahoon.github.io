Background#

Many services on my local network run directly over IP, which poses a security threat that cannot be ignored. In an isolated environment, it is important to set up a local Root CA for the LAN.

The tool used is very simple => mkcert

About mkcert#

mkcert is a zero-configuration local CA solution.

Open source project: https://github.com/FiloSottile/mkcert

Environment#

Use my desktop as the hosting server, it is Ubuntu 26.04, IP: 192.168.1.4

Install and Setup#

sudo apt install libnss3-tools mkcert
// on fedora it's the nss-tools and mkcert packages

Create local CA certs#

❯ mkcert --install
Created a new local CA 💥
The local CA is now installed in the system trust store! ⚡️
The local CA is now installed in the Firefox and/or Chrome/Chromium trust store (requires browser restart)! 🦊
// You can see it is imported into the local host

Create my first cert#

My machines are all accessed directly via IP. The server IP is 192.168.1.4, so my certificate is also created directly based on the IP.

// vault.pem is the cert file
// vault-key.pem is the private key file
❯ mkcert -cert-file vault.pem -key-file vault-key.pem 192.168.1.4 localhost 127.0.0.1

Created a new certificate valid for the following names 📜
 - "192.168.1.4"
 - "localhost"
 - "127.0.0.1"

The certificate is at "vault.pem" and the key at "vault-key.pem" ✅

It will expire on 27 August 2028 🗓

Check root CA#

We often need to import the root CA into client machines so they won’t complain when accessing HTTPS websites signed by this CA.

// get the default dir of root CA
❯ mkcert -CAROOT
/home/yahoon/.local/share/mkcert

❯ cat /home/yahoon/.local/share/mkcert/rootCA
rootCA-key.pem  rootCA.pem  
// rootCA.pem is the pub cert to be shared
// rootCA-key.pm is private, Never should be shared.

Import to Client Host#

Take a Fedora client as an example:

  1. Copy the content of rootCA.pem
  2. Save it to /etc/pki/ca-trust/source/anchors/rootca.pem on Fedora
  3. Run ❯ sudo update-ca-trust
