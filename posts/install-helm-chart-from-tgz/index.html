Offline Installation of Helm Chart (tar.gz File)#

Recently I’ve been learning Helm. As an operations engineer, I haven’t had much exposure to deploying applications on K8s, so I decided to get some hands-on practice.

Using the official podinfo chart, the steps are as follows:

Step 1: Pull the Chart#

The chart package itself is a tar.gz archive.

First, there are two channels for publishing chart packages:

  • HTTP repo (e.g., Artifact Hub)
  • OCI address (oci://ghcr.io/..., similar to Docker Hub) — published by the chart author to an OCI registry Which one to use depends on the author’s preference.

Learning the helm pull command

❯ helm pull --help

Retrieve a package from a package repository, and download it locally.

Usage:  helm pull [chart URL | repo/chartname] [...] [flags]

There are also two ways to pull a chart locally (depending on where the chart is located):

  1. Pull from OCI registry

The oci:// address is usually provided by the author.

helm pull oci://ghcr.io/stefanprodan/charts/podinfo --version 6.11.2
  1. Pull from HTTP repo

First, get the chart name and repo.

❯ helm search hub podinfo --list-repo-url
URL                                               	CHART VERSION	APP VERSION	DESCRIPTION                               	REPO URL              https://artifacthub.io/packages/helm/podinfo/po...	6.13.0       	6.13.0     	Podinfo Helm chart for Kubernetes         	https://stefanprodan.github.io/podinfo
https://artifacthub.io/packages/helm/flagger/po...	6.1.4        	6.1.3      	Flagger canary deployment demo application	https://flagger.app  
// The first URL is the webpage (truncated), the following is the REPO URL
// Issue: the chart name isn't visible here

// So view the full info
❯ helm search hub podinfo --list-repo-url -o yaml
- app_version: 6.13.0
  description: Podinfo Helm chart for Kubernetes
  repository:
    name: podinfo
    url: https://stefanprodan.github.io/podinfo   // This is the repo URL
  url: https://artifacthub.io/packages/helm/podinfo/podinfo  // This is the chart webpage
  version: 6.13.0

Note the chart webpage URL: extract the chart name from it.

https://artifacthub.io/packages/helm/podinfo/podinfo
                                        ↑       ↑
                                      repo    chart name

Now you can view the specific chart info.

❯ helm show chart podinfo --repo https://stefanprodan.github.io/podinfo
apiVersion: v1
appVersion: 6.13.0
description: Podinfo Helm chart for Kubernetes
home: https://github.com/stefanprodan/podinfo
kubeVersion: '>=1.23.0-0'
maintainers:
- email: stefanprodan@users.noreply.github.com
  name: stefanprodan
name: podinfo
sources:
- https://github.com/stefanprodan/podinfo
version: 6.13.0

Now pull it.

// pull the chart pkg file
❯ helm pull podinfo --repo https://stefanprodan.github.io/podinfo --version 6.13.0

// The file is now available locally
❯ ls
podinfo-6.13.0.tgz

You can extract the tgz file to see its contents.

Step 2: Install tar.gz Locally#

Note that I named the release yahoon-podinfo2 after deploying the podinfo chart to K8s.

# Install directly using the local .tgz file, no internet required
❯ helm install yahoon-podinfo2 ./podinfo-6.13.0.tgz

Step 3: Test Access#

Run the following command in the terminal (do not close it): ❯ kubectl -n default port-forward deploy/yahoon-podinfo2 8080:9898

Open http://127.0.0.1:8080 in your browser.

Then go back to the terminal and exit the previous command.

Step 4: Delete the Deployed Release#

❯ helm ls
NAME       	NAMESPACE	REVISION	UPDATED                                	STATUS  	CHART         	APP VERSION
yahoon-podinfo2	default  	1       	2026-06-09 10:55:09.61677018 +0800 CST 	deployed	podinfo-6.13.0	6.13.0     

❯ helm uninstall yahoon-podinfo2
