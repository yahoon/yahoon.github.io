Installing the Password Management Server VaultWarden#

Background#

I’ve been using Bitwarden as my password manager, but some issues have gradually emerged:

  • Access from China is always slow, probably because the servers are overseas
  • What if one day it gets blocked?

After some comparison and research, I chose to self-host VaultWarden

Why VaultWarden?#

  • Cost of self-hosting Bitwarden: Although the official Bitwarden supports self-hosting, its backend is written in .NET and is very memory-hungry (typically requires 2GB~4GB+ RAM), which is unfriendly for my PC. Vaultwarden, on the other hand, is extremely lightweight, requiring only tens of MB of RAM to run smoothly.
  • Compatibility: Vaultwarden is fully compatible with Bitwarden’s official apps and browser extensions. You just need to fill in your self-hosted URL in the “Self-hosted URL” field when logging in on the client side.
  • Deployment: Vaultwarden supports one-click deployment with Podman / Docker, while Bitwarden requires Docker Compose.

Introduction#

Project page: https://github.com/dani-garcia/vaultwarden

Fully open source, very simple to deploy.

Supports access via web, Bitwarden’s official apps, and browser extensions.

Hosting Machine#

I will use my desktop as the hosting server (OS: Ubuntu 26.04, IP: 192.168.1.4)

Deployment Process#

Step 0: Prepare SSL Certificates#

VaultWarden only works under HTTPS, so you need to prepare server certificates first.

I already created the certs in my last post.

> mkdir /home/yahoon/srv/vaultwarden/{data,ssl}

// move the key files
❯ mv vault.pem vault-key.pem srv/vaultwarden/ssl/
❯ ls srv/vaultwarden/ssl/
vault-key.pem  vault.pem

Step 1: Prepare the Image#

On the client machine:

// Pull the image with proxy enabled
❯ podman pull docker.io/vaultwarden/server:latest

// Tag and push to local registry
❯ podman tag docker.io/vaultwarden/server:latest 192.168.1.4:5000/vaultwarden/server:latest

❯ podman push 192.168.1.4:5000/vaultwarden/server:latest

Step 2: Start Vaultwarden#

podman run -d \
  --name vaultwarden \
  -v /home/yahoon/srv/vaultwarden/data:/data:Z \
  -v /home/yahoon/srv/vaultwarden/ssl:/ssl:Z \
  -e ROCKET_TLS='{certs="/ssl/vault.pem",key="/ssl/vault-key.pem"}' \
  -p 8000:80 \
  --restart unless-stopped \
  192.168.1.4:5000/vaultwarden/server:latest

❯ podman ps
278bb12c867f  192.168.1.4:5000/vaultwarden/server:latest  /start.sh             9 minutes ago  Up 9 minutes         0.0.0.0:8000->80/tcp                                               vaultwarden

// Test locally
❯ curl -v https://127.0.0.1:8000
// Confirm content is returned

// Open firewall
❯ sudo ufw allow 8000/tcp
❯ sudo ufw status 
Status: active
To                         Action      From
--                         ------      ----
8000/tcp                   ALLOW       Anywhere

Usage#

First, import the internal root CA

Using a Fedora client host as an example:

  1. Copy the contents of rootCA.pem on the Ubuntu server to the Fedora machine
  2. Save it to /etc/pki/ca-trust/source/anchors/rootca.pem
  3. Run ❯ sudo update-ca-trust

Open a browser and visit

Open https://192.168.1.4:8000/#/vault, create an account and you’re ready to go.
