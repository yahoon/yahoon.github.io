Send mail via Public SMTP by CLI#

I needed to configure an alerting feature that sends emails from my local machine (Fedora 44) to a public email server via the command line.

Installing Postfix locally is not a good choice, because public email services all perform strict checks to prevent spam.

Plan: I have a 126.com email account, so I’ll use 126.com’s SMTP server.

Step 1: Get your 126.com Authorization Code#

For security reasons, 126.com will not let you use your regular login password in the terminal. You need an app-specific password.

  1. Log in to your 126.com webmail.

  2. Go to Settings -> POP3/SMTP/IMAP.

  3. Enable the POP3/SMTP service switch.

  4. Follow the prompts to generate an Authorization Code. Copy this code; you will use it as your password.

Step 2: Install msmtp and mailx on Fedora 44#

Open your terminal and install msmtp (the SMTP client) and s-nail (the modern mailx user interface for sending the mail body):

sudo dnf install msmtp s-nail

Step 3: Configure msmtp#

Create a configuration file in your user directory:

mkdir -p ~/.config/msmtp
vi ~/.config/msmtp/config

Paste the following configuration, replacing the placeholders with your actual 126 email and the authorization code you copied in Step 1:

# Set default values for all further accounts.
defaults
auth             on
tls              on
tls_starttls     off
tls_certcheck    on
tls_trust_file   /etc/ssl/certs/ca-certificates.crt
logfile          ~/.config/msmtp/msmtp.log

# 126.com Account
account          my_account
host             smtp.126.com
port             465
from             abc@126.com
user             abc@126.com
password         <the authorization code>

# Set a default account
account default : my_account

Set its file mode to 0600

Step 4: Configure the mail command link#

Tell s-nail (the mail command) to use msmtp as the backbone sender.

Open your bash/zsh configuration file: ~/.bashrc, add the following line

alias mail='s-nail -S mta=/usr/bin/msmtp'

Apply it

source ~/.bashrc

Step 5: Test Sending a Mail 🚀#

Now you are completely set up! Test it by running this single-line command:

❯ echo "This is the body of my email sent from Fedora CLI." | mail -s "Fedora Terminal Test" 1122233@qq.com

Result#

  • The sent email can be seen in 126.com’s outbox.
  • The received email can be seen in QQ’s inbox.
