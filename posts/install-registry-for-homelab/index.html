Setup Container Registry for Home Lab

Introduction#

I’m building my home lab. In corporate environments, various services are ready to use out of the box, so I’ve never had to worry about services being blocked or slow network speeds.

However, in a home lab network environment, many websites are inaccessible, or it’s hard to find free alternatives. The most typical example is Docker Hub. So I had to set up my own.

My requirements are simple:

  • Only for use within the home lab intranet
  • Any machine on the intranet can push and pull images
  • HTTP access is sufficient (no HTTPS needed)
  • No authentication required

Server#

My desktop:

  • OS: Ubuntu 26.04
  • IP: 192.168.1.4
  • Exposed host port: 192.168.1.4:5000
  • Current regular user: yahoon (with sudo privileges)

Steps#

  1. Create a persistent data directory

Store registry data under the current user’s srv/registry directory:

❯ cd /home/yahoon/srv

// Store data on a dedicated data disk
❯ mkdir /home/yahoon/mnt/registry

❯ ln -s /home/yahoon/mnt/registry/ .
> ll
lrwxrwxrwx  1 yahoon yahoon   registry@ -> /home/yahoon/mnt/registry/
  1. Open port 5000/tcp on the firewall
> sudo ufw allow 5000/tcp
> sudo ufw reload
  1. Start the Registry Container

Name the container local-registry:

// This requires a proxy to be enabled
> podman run -d \
  -p 5000:5000 \
  --name local-registry \
  -v /home/yahoon/srv/registry:/var/lib/registry \
  docker.io/library/registry:2
// Since it's Ubuntu, the :Z suffix after -v is not needed; that's only for SELinux

> podman ps
CONTAINER ID  IMAGE                                  COMMAND               CREATED        STATUS                 PORTS                                                              NAMES
9019e58b7e71  docker.io/library/registry:2           /etc/docker/regis...  3 seconds ago  Up 3 seconds           0.0.0.0:5000->5000/tcp                                             local-registry
  1. Enable User Process Lingering

On Ubuntu / RHEL / Fedora, when a regular user yahoon disconnects their SSH session (logs out), the system will by default reclaim all background processes for that user (containers will also be suspended). Run the following command to allow services for the yahoon user to continue running in the background even when not logged in:

sudo loginctl enable-linger yahoon
  1. (Optional) Configure the registry container to start on boot

This can be achieved using systemd.

Since my registry container runs under a personal user account, we need to use systemd’s --user mode to manage the container so it starts automatically after a system reboot.

# 1. Create the systemd configuration directory for user yahoon
mkdir -p ~/.config/systemd/user/

# 2. Generate the systemd service configuration file
podman generate systemd --new --name local-registry > ~/.config/systemd/user/container-local-registry.service

# 3. Reload the systemd daemon for user yahoon
systemctl --user daemon-reload

# 4. Enable the service and start it immediately
systemctl --user enable --now container-local-registry.service

Client Configuration#

Using Fedora 43 and RHEL 9 clients as examples:

Edit the file /etc/containers/registries.conf:

> sudo vi /etc/containers/registries.conf
// Append the following content at the end
[[registry]]
location = "192.168.1.4:5000"
insecure = true

Client Operations#

Test pull and push

// Pull a small image (using a domestic mirror)
❯ podman pull alpine 

❯ podman images |grep alpine
docker.io/library/alpine       latest        3cb067eab609   8.74 MB

❯ podman tag docker.io/library/alpine 192.168.1.4:5000/alpine:latest

// Test: push
❯ podman push 192.168.1.4:5000/alpine:latest

// Test: pull with another user
// Switch to root with sudo
# podman pull 192.168.1.4:5000/alpine:latest

Test with curl and skopeo

# curl http://192.168.1.4:5000/v2/_catalog
{"repositories":["alpine"]}

# curl http://192.168.1.4:5000/v2/alpine/tags/list
{"name":"alpine","tags":["latest"]}

# skopeo list-tags docker://192.168.1.4:5000/alpine
{
    "Repository": "192.168.1.4:5000/alpine",
    "Tags": [
        "latest"
    ]
}
