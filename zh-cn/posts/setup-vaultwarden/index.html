安装密码管理服务器 VaultWarden#

背景#

我一直使用bitwarden web 作为我的密码管理器，但是有些问题逐渐浮现：

  • 国内访问一直比较慢，可能是它的服务器在国外的原因
  • 万一哪天被禁止访问了，该怎么办

经过一番比较和研究，我选择内部部署 VaultWarden

为什么是它？ VaultWarden!#

  • 自己部署bitwarden的成本: 虽然官方 Bitwarden 支持私人部署，但是由于它本身后端是用 .NET 写的，非常吃内存（通常需要 2GB~4GB RAM 以上），对我自己PC来说极不友好。而 Vaultwarden 极其轻量，只需要几十 MB 的内存就能流畅运行。
  • 兼容性: Vaultwarden完全兼容 Bitwarden 的官方 App 和浏览器插件。你只需要在客户端登录时，将“自定域（Self-hosted URL）”填写为你自己搭建的网址即可。
  • 部署方式: Vaultwarden支持podman / Docker 一键部署， 而bitwarden需要的是docker/docker compose。

介绍#

项目主页： https://github.com/dani-garcia/vaultwarden

完全开源， 部署非常简单。

支持网页，Bitwarden 的官方 App 和浏览器插件访问。

宿主机#

我将以我的台式机作为宿主机（OS: Ubuntu 26.04, IP: 192.168.1.4）

部署过程#

Step 0: 准备SSL 证书#

VaultWarden 只有在HTTPS下才工作， 所以要先准备好服务器证书

在上一篇文章中我已经创建好了证书。

> mkdir /home/yahoon/srv/vaultwarden/{data,ssl}
 
// move the key files
❯ mv vault.pem vault-key.pem srv/vaultwarden/ssl/
❯ ls srv/vaultwarden/ssl/
vault-key.pem  vault.pem

Step 1: 准备镜像#

在客户机上：

// 在开代理的时候拉镜像
❯ podman pull docker.io/vaultwarden/server:latest

// 推送到本地 registry
❯ podman tag docker.io/vaultwarden/server:latest 192.168.1.4:5000/vaultwarden/server:latest

❯ podman push 192.168.1.4:5000/vaultwarden/server:latest

Step 2: 启动vaultwarden#

podman run -d \
  --name vaultwarden \
  -v /home/yahoon/srv/vaultwarden/data:/data:Z \
  -v /home/yahoon/srv/vaultwarden/ssl:/ssl:Z \
  -e ROCKET_TLS='{certs="/ssl/vault.pem",key="/ssl/vault-key.pem"}' \
  -p 8000:80 \
  --restart unless-stopped \
  192.168.1.4:5000/vaultwarden/server:latest

❯ podman ps
278bb12c867f  192.168.1.4:5000/vaultwarden/server:latest  /start.sh             9 minutes ago  Up 9 minutes         0.0.0.0:8000->80/tcp                                               vaultwarden

// 本机测试
❯ curl -v https://127.0.0.1:8000
// 确认确实有内容返回

// 打开防火墙
❯ sudo ufw allow 8000/tcp
❯ sudo ufw status 
Status: active
To                         Action      From
--                         ------      ----
8000/tcp                   ALLOW       Anywhere

使用#

首先导入内网root CA

以fedora的客户机为例

  1. 先在Ubuntu机器上复制 rootCA.pem 的内容
  2. 存到fedara中的 /etc/pki/ca-trust/source/anchors/rootca.pem文件
  3. 运行 ❯ sudo update-ca-trust

打开浏览器访问

打开 https://192.168.1.4:8000/#/vault, 创建账号即可访问
