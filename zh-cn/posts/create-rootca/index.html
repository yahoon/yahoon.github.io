为内网创建Root CA#

因为我的内网的很多服务都是直接用IP运行的，这个带来的安全威胁不容忽视。在一个隔离的环境中，建立本地局域网的local Root CA非常重要。

使用的工具也非常简单=> mkcert

介绍 mkcert#

mkcert 是一个0配置的本地CA 方案。

开源项目： https://github.com/FiloSottile/mkcert

安装环境#

我把我的台式机作为宿主服务器（Ubuntu 26.04）,IP: 192.168.1.4

安装与部署#

sudo apt install libnss3-tools mkcert
// fedora上是nss-tools mkcert 这2个包

创建本地root CA#

❯ mkcert --install
Created a new local CA 💥
The local CA is now installed in the system trust store! ⚡️
The local CA is now installed in the Firefox and/or Chrome/Chromium trust store (requires browser restart)! 🦊
// 可见它已经导入了本机

创建第一个证书#

我的机器都是用IP直接访问的，服务器的IP是 192.168.1.4。 所以我的证书也是直接基于IP创建的

// vault.pem 是 证书文件
// vault-key.pem 是 私钥文件
❯ mkcert -cert-file vault.pem -key-file vault-key.pem 192.168.1.4 localhost 127.0.0.1

Created a new certificate valid for the following names 📜
 - "192.168.1.4"
 - "localhost"
 - "127.0.0.1"

The certificate is at "vault.pem" and the key at "vault-key.pem" ✅

It will expire on 27 August 2028 🗓

查看 root CA#

我们经常需要把root ca导入到客户机器，这样它们在访问这个CA 签发https网站时就不会报错了

// 获得本地 root CA的目录
❯ mkcert -CAROOT
/home/yahoon/.local/share/mkcert

❯ cat /home/yahoon/.local/share/mkcert/rootCA
rootCA-key.pem  rootCA.pem  
// rootCA.pem 是可以被共享的pub 证书
// rootCA-key.pm 是私钥，必须保密，永不泄漏

导入到客户端机器#

以Fedora的客户机为例

  1. 复制 rootCA.pem 的内容
  2. 存到Fedara中的 /etc/pki/ca-trust/source/anchors/rootca.pem文件
  3. 运行 ❯ sudo update-ca-trust
