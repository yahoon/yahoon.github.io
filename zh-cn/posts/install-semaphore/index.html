背景#

我是一名 Ansible Automation Platform （AAP）的资深用户，深切感受到一个自动化平台的重要性。作为AAP的上游， AWX 这个开源项目已经许久没有更新，最新版本还停留在 2024年.

这让我不得不寻求其他的替代工具。

通过AI 搜索，Semaphore UI 脱颖而出。 它是一个类AAP的自动化运维平台， 有Web UI 且不光支持ansible, 还有terraform等。

官网： https://semaphoreui.com/

开源项目： https://github.com/semaphoreui/semaphore

宿主服务器#

我将我的台式机部署服务器

OS: Ubuntu 26.04 ， 已经安装有 podman

Semaphore#

Install#

根据github主页推荐， 使用容器安装

若要下载其他格式，可参考：

  • 在主页上有安装向导 https://semaphoreui.com/install
  • 下载各种包的地址： https://github.com/semaphoreui/semaphore/releases

打开 Docker安装引导页面 https://semaphoreui.com/install/docker/2_18 ， 填入定制化的选项，它就会自动生成命令

它给我的命令是

docker run --name semaphore \
-p 3000:3000 \
-e SEMAPHORE_DB_DIALECT=sqlite \
-e SEMAPHORE_ADMIN=admin \
-e SEMAPHORE_ADMIN_PASSWORD=admin \
-e SEMAPHORE_ADMIN_NAME="Admin" \
-e SEMAPHORE_ADMIN_EMAIL=semaphore-admin@yahoon.io \
-v semaphore_data:/var/lib/semaphore \
-v semaphore_config:/etc/semaphore \
-v semaphore_tmp:/tmp/semaphore \
-d semaphoreui/semaphore:v2.18.0

准备semaphore 镜像#

因为是homelab, 访问不了docker hub，所以我要先在开代理的模式下将以下镜像拉到本地

docker.io/semaphoreui/semaphore        latest      30f8744c37ca  2 days ago   944 MB

以防万一，推送到本地的registry

❯ podman tag docker.io/semaphoreui/semaphore:latest 192.168.1.4:5000/semaphoreui/semaphore:latest

❯ podman push 192.168.1.4:5000/semaphoreui/semaphore:latest

启动#

// 以下命令在非代理的Terminal 环境中执行

// 修整后的命令是:
podman run --name semaphore \
-p 3000:3000 \
-e SEMAPHORE_DB_DIALECT=sqlite \
-e SEMAPHORE_ADMIN=admin \
-e SEMAPHORE_ADMIN_PASSWORD=admin \
-e SEMAPHORE_ADMIN_NAME="Admin" \
-e SEMAPHORE_ADMIN_EMAIL=semaphore-admin@yahoon.io \
-v semaphore_data:/var/lib/semaphore \
-v semaphore_config:/etc/semaphore \
-v semaphore_tmp:/tmp/semaphore \
-d semaphoreui/semaphore:latest

// 我把命令改成了docker -》 podman， 镜像版本 v2.18.0 -> latest

❯ podman ps

// 本机测试
❯ curl -v http://127.0.0.1:3000

访问#

  1. 从客户机的浏览器访问 http://192.168.1.4:3000
  2. 输入 admin的用户名和密码登录
  3. 进去以后，根据提示创建demo project
  4. 进到 Task templates里面去执行一些task
