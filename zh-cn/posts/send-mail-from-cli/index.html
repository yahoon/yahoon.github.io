通过命令行使用公网SMTP服务器发邮件#

我需要配置一个报警功能，需要从本机( Fedora 44)的命令行去往公网的邮箱发送email.

在本机装posftix并不是一个好的选择，因为公网的邮件服务都会为了预防垃圾邮件而做严格检查。

计划： 我有126.com的邮箱，所以用126.com的SMTP服务器

步骤一：获取你的 126.com 授权码#

出于安全原因，126.com 不会允许你在终端中使用常规登录密码，你需要一个应用专用密码。

  1. 登录你的 126.com 网页邮箱。

  2. 进入 设置 -> POP3/SMTP/IMAP。

  3. 开启 POP3/SMTP服务 开关。

  4. 按照提示生成一个授权码。复制此代码，你将用它作为密码。

步骤二：在 Fedora 44 上安装 msmtp 和 mailx#

打开终端，安装 msmtp（SMTP 客户端）和 s-nail（现代 mailx 用户界面，用于发送邮件正文）：

sudo dnf install msmtp s-nail

步骤三：配置 msmtp#

在你的用户目录下创建配置文件：

mkdir -p ~/.config/msmtp
vi ~/.config/msmtp/config

粘贴以下配置，将占位符替换为你实际的 126 邮箱和步骤一中复制的授权码：

# Set default values for all further accounts.
defaults
auth             on
tls              on
tls_starttls     off
tls_certcheck    on
tls_trust_file   /etc/ssl/certs/ca-certificates.crt
logfile          ~/.config/msmtp/msmtp.log

# 126.com Account
account          my_account
host             smtp.126.com
port             465
from             abc@126.com
user             abc@126.com
password         <the authorization code>

# Set a default account
account default : my_account

将其文件权限设置为 0600

步骤四：配置 mail 命令链接#

告诉 s-nail（即 mail 命令）使用 msmtp 作为后端发送程序。

打开你的 bash/zsh 配置文件 ~/.bashrc，添加以下行：

alias mail='s-nail -S mta=/usr/bin/msmtp'

使其生效：

source ~/.bashrc

步骤五：测试发送邮件 🚀#

现在你已经完全配置好了！运行下面这行命令来测试：

❯ echo "This is the body of my email sent from Fedora CLI." | mail -s "Fedora Terminal Test" 112233@qq.com

效果#

  • 在126的发件箱里面看到发出的邮件
  • 在qq的收件箱里面看到受到的邮件
