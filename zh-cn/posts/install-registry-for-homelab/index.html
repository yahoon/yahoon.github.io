为Home Lab搭建Container Registry

介绍#

我正在构建我的Home lab。 以前在公司各种服务都是开箱即用，我根本不会考虑某个服务被墙或者网速慢的问题。

可是home lab的网络环境很多网站不可以访问，或者说找不到免费的替代品, 最典型的就是docker hub。 所有只有自己搭建了。

我的需求很简单：

  • 只给home lab 内网使用
  • 内网的任意机器可以push 和pull 镜像
  • http访问就行 (without https)
  • 认证也不需要

服务器#

My desktop:

  • OS: Ubuntu 26.04
  • IP: 192.168.1.4
  • 暴露主机端口： 192.168.1.4:5000
  • 当前普通用户 yahoon (有sudo权限)

Steps#

  1. 创建数据的持久化目录

registry data 放在当前用户 srv/registry 下

❯ cd /home/yahoon/srv

// 数据放在专有的数据盘上
❯ mkdir /home/yahoon/mnt/registry

❯ ln -s /home/yahoon/mnt/registry/ .
> ll
lrwxrwxrwx  1 yahoon yahoon   registry@ -> /home/yahoon/mnt/registry/
  1. 在防火墙上放开 5000/tcp 端口
> sudo ufw allow 5000/tcp
> sudo ufw reload
  1. 启动 Registry 容器

我们给容器起名字为local-registry:

// 这个需要打开代理
> podman run -d \
  -p 5000:5000 \
  --name local-registry \
  -v /home/yahoon/srv/registry:/var/lib/registry \
  docker.io/library/registry:2
// 因为是Ubuntu, 所以-v后面不需要:Z, 那是只适用于Selinux

> podman ps
CONTAINER ID  IMAGE                                  COMMAND               CREATED        STATUS                 PORTS                                                              NAMES
9019e58b7e71  docker.io/library/registry:2           /etc/docker/regis...  3 seconds ago  Up 3 seconds           0.0.0.0:5000->5000/tcp                                             local-registry
  1. 开启用户进程常驻

在 Ubuntu / RHEL / Fedora 中，如果普通用户 yahoon 断开了 SSH 连接（登出），系统默认会回收该用户的所有后台进程（容器也会随之挂起）。执行以下命令，允许 yahoon 用户的服务在不登录时依然在后台保持运行：

sudo loginctl enable-linger yahoon
  1. (可选） 配置 registry容器开机自动启动

可以通过Systemd 实现registry容器的开机自启。

由于我的registry是以个人用户运行，我们需要利用 systemd 的 --user 模式来管理容器，使其在系统重启后自动运行。

# 1. 创建属于 yahoon 用户的 systemd 配置目录
mkdir -p ~/.config/systemd/user/

# 2. 生成 systemd 服务配置文件
podman generate systemd --new --name local-registry > ~/.config/systemd/user/container-local-registry.service

# 3. 刷新 yahoon 用户的 systemd 守护进程
systemctl --user daemon-reload

# 4. 启用该服务并立即启动
systemctl --user enable --now container-local-registry.service

客户端配置#

以 Fedora 43 和 RHEL 9 的客户端为例

编辑文件 /etc/containers/registries.conf ：

> sudo vi /etc/containers/registries.conf
// 在末尾增加如下内容
[[registry]]
location = "192.168.1.4:5000"
insecure = true

客户端操作#

测试pull 和push

// push a small image (国内加速)
❯ podman pull alpine 

❯ podman images |grep alpine
docker.io/library/alpine       latest        3cb067eab609   8.74 MB

❯ podman tag docker.io/library/alpine 192.168.1.4:5000/alpine:latest

// test: push
❯ podman push 192.168.1.4:5000/alpine:latest

// test: pull
// sudo 到root下
# podman pull 192.168.1.4:5000/alpine:latest

通过curl 和 skopeo 来测试

# curl http://192.168.1.4:5000/v2/_catalog
{"repositories":["alpine"]}

# curl http://192.168.1.4:5000/v2/alpine/tags/list
{"name":"alpine","tags":["latest"]}

# skopeo list-tags docker://192.168.1.4:5000/alpine
{
    "Repository": "192.168.1.4:5000/alpine",
    "Tags": [
        "latest"
    ]
}
